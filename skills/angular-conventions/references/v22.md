# Angular v22

The current stable major (released 2026-06-03; latest patch 22.0.6). This is the release the skill's forward-looking idioms target, so the rest of SKILL.md applies as written - this file adds the v22-specific runtime, the APIs that reached stable here, and the one deprecation that changes wiring. Load it only if the workspace is on v22. Fact-checked against angular.dev.

## Runtime

- Node.js `^22.22.3 || ^24.15.0 || ^26.0.0`, TypeScript `>=6.0.0 <6.1.0` - **v22 is the first to require TypeScript 6.x**. RxJS `^6.5.3 || ^7.4.0`.
- Support: active (current major).

## Reached stable in v22 - safe to use as the default now

- The whole resource family - `resource`, `rxResource`, `httpResource` - graduated from experimental. `httpResource(() => ...)` is the lean default for reactive HTTP reads.
- **Signal Forms** (`form()` from `@angular/forms/signals`) is stable and the recommended approach for new forms - with `validate` (cross-field via `valueOf(path)`), `validateAsync` / `validateHttp`, and `validateStandardSchema` (Zod / Valibot). Below v22 all of this was experimental.
- `@angular/aria` headless a11y directives lost the developer-preview tag. The pattern set is ~12 by v22 - check `angular.dev/guide/aria` rather than assuming a fixed roster.

## Changed wiring in v22 - the one live gotcha

- **Incremental hydration is on by default** with plain `provideClientHydration()`. `withIncrementalHydration()` is **deprecated** (removal targeted v24) - opt *out* with `provideClientHydration(withNoIncrementalHydration())`. It still auto-enables event replay, so remove any `withEventReplay()` you carried over.

## Reported for v22 - confirm before relying

The official `angular.dev/events/v22` page confirms the stabilizations above but states other changes only as generic 'template enhancements' and 'API improvements'. These come from secondary summaries and are not yet pinned to an official page - do not assert exact syntax without checking the v22 release notes:

- `OnPush` as the default change-detection strategy for new apps (this codebase mandates `OnPush` regardless, so nothing changes here).
- Template spread/rest syntax, stacked `@switch` with exhaustive `@default never`, inline arrow functions in event bindings, and a `@Service` decorator.
